# クロック・アプリ 仕様書

## デバイス仕様
- **ターゲットデバイス**: AQUOS sense4 plus (2400×1080)
- **対応ブラウザ**: Chrome 等のモダンブラウザ（PWA対応必須）

## 表示・レイアウト仕様

### 共通仕様
- **フォント**: Google Fonts `M PLUS 2` を全面的に採用。
- **解像度非依存の設計**: 端末の画素密度に左右されず、指定したアスペクト比（横長 2400:1080 / 縦長 1080:2400）で画面いっぱいにストレッチされる自動スケーリング機能を搭載。
- **点滅インジケーター**: 時刻と日付の間に、1秒周期で明滅するドットインジケーターを配置。動作確認の視覚的なフィードバックを提供。

### 横長表示 (Landscape) - 基本
- **レイアウト**: 左右2段組。
- **左側ブロック**: 時刻表示（HH:MM）。文字色 `#eee`。
- **右側ブロック**: 
  - 上段：西暦 / 元号 / 月の英略称 / 曜日の英略称。
  - 下段：月 / 日 / 曜日。

### 縦長表示 (Portrait)
- **レイアウト**: 上下4行構成（時刻 / 年号情報 / 月日情報 / 天候情報）。
- **サイズ最適化**: 縦長画面を占有するよう、文字サイズと各要素間の余白（gap）を動的に調整。

## PWA / ウェブ機能
- **インストール対応**: Web App Manifest 対応。ホーム画面に追加してスタンドアロンで動作。
- **オフライン動作**: Service Worker による全アセットのキャッシュ。
- **スリープ防止 (Wake Lock)**: 全画面表示中、またはPWA起動中に画面が消灯するのを防止（Screen Wake Lock API）。
- **照度センサー連動**: 照度が 10 lux 未満になると、自動的に輝度を 60% に抑制（`.dim` モード）。

## 設定・カスタマイズ仕様

### 設定管理
- **データ永続化**: すべての設定値はブラウザの `localStorage` に保存され、再起動後も維持。
- **初期誘導**: APIキーが設定されていない場合、起動時に自動的に設定画面を表示してセットアップを促す。
- **デフォルト値**: 利便性のため、初期状態で動作可能なAPIキーとロケーション（札幌）を設定済み。

### 設定画面項目
- **天気設定**: 
  - WeatherAPI APIキーの入力。
  - 取得場所（Location）の入力。
- **表示設定**: 
  - 液晶タイプ（液晶 / 有機EL）の切り替え。
- **クレジット**: 
  - プロジェクトリポジトリおよび WeatherAPI.com の規定クレジットを表示。リンクは外部ブラウザで起動。

### ハードウェア保護 (OLED対応)
- **Pixel Shifting**: 表示タイプが「有機EL」に設定されている場合、1分ごとにメイン表示位置を数ピクセル単位でランダムに微動させ、焼き付きを防止。

## 技術仕様
- **フレームワーク**: なし（Vanilla JS / CSS）。
- **外部通信**: WeatherAPI.com より天候情報を取得。
- **通信節約 (Cache)**: 取得に成功したデータを保存し、30分以内はリクエストを行わずキャッシュを表示。取得失敗時は `----` を表示。
- **操作UI**: ⛶ (Fullscreen) および ⚙ (Settings) ボタンを 100px のタップしやすいサイズで固定配置。
