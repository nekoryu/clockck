# clockck 仕様書

## 動作環境・ターゲット

- **ターゲットデバイス**: スマートフォン全般。主に AQUOS sense4 plus での使用を想定（解像度 2400×1080、ディスプレイ 液晶 (LED)）
- **対応ブラウザ**: Chrome 等のモダンブラウザ（PWA 対応必須）
- **利用形態**: Web App Manifest と Service Worker によるインストール（スタンドアロン起動推奨）

---

## 全体レイアウト・UI 仕様

### フォント

Google Fonts `M PLUS 2` を全面採用。

### 解像度非依存の自動スケーリング

ユーザーが設定したアスペクト比を基準解像度として `<main>` 要素のサイズを JS で動的に確定し、`transform: scale()` によって画面に収まるよう自動スケーリングを行う。

- **短辺 1080px 固定**。長辺はアスペクト比設定に従い決定（後述）。
- スケール係数は `min(viewWidth / baseWidth, viewHeight / baseHeight)` で算出（均一スケール・アスペクト比を厳守し、はみ出しゼロを保証）。
- 余白は `translate` によって中央揃えで配置される。
- `resize` / `orientationchange` イベントで追従再計算。

### 表示方向の最適化

| 向き | レイアウト |
|------|-----------|
| **横長 (Landscape)** | 時刻（左）と年月日（右）を横並び配置。下部に天候情報 |
| **縦長 (Portrait)** | 時刻 → 年月日 → 天候情報を上から下にブロック配置 |

## 表示項目

### 時刻

- 時・分を大型フォントで表示（横: 340px、縦: 320px）
- 秒（任意表示）

### 年月日

- 西暦年
- 和暦（`Intl.DateTimeFormat` の `ja-JP-u-ca-japanese` を使用）
- 月名・曜日（英語略称）
- 日付（`MM/DD (曜)`）

### 天候情報

データソース: [WeatherAPI.com](https://www.weatherapi.com/)

| 項目 | アイコン | デフォルト表示 |
|------|----------|---------------|
| 天候状況（テキスト） + 絵文字 | 天候テキストに応じて自動選択（☂ ☀ ☁ ❄ ⚡ 🌫） | 常時表示（天候エリアが有効な場合） |
| 気温 | 🌡 | 常時表示 |
| 体感温度 | — | OFF |
| 湿度 | 💧 | OFF |
| 風速 | 💨 | OFF |
| UV 指数 | ☀ | OFF |
| 降水量 | ☔ | OFF |
| 気圧 | ⏲ | OFF |

- 風速単位: km/h → m/s 換算（小数第1位）
- キャッシュ: 取得した生データ（JSON）を `localStorage` に保存。30 分以内はキャッシュを利用して API 通信を節約。
- 設定保存時に強制再取得（`force=true`）。

## 設定機能

画面右上の ⚙ アイコンから設定モーダルを開閉。すべての設定は `localStorage` に保存され、次回起動時に復元される。

### localStorage キー一覧

| キー | 内容 | デフォルト値 |
|------|------|-------------|
| `clockck_display_type` | 液晶 / 有機EL | `lcd` |
| `clockck_aspect_ratio` | アスペクト比 | `16:9` |
| `clockck_theme` | カラーテーマ | `auto` |
| `clockck_visible_items` | 各項目の表示ON/OFF（JSON） | sec:ON, weather:ON, 他:OFF |
| `clockck_api_key` | WeatherAPI APIキー | （ビルトイン値） |
| `clockck_location` | 取得場所（都市名 or 座標） | `Sapporo` |
| `clockck_weather_cache` | 天気データキャッシュ（JSON） | — |
| `clockck_weather_cache_time` | キャッシュ取得時刻（Unix ms） | — |

### 表示設定

#### 液晶タイプ

| 値 | 説明 |
|----|------|
| `lcd` | 液晶（位置固定） |
| `oled` | 有機EL（焼き付き防止モード） |

#### アスペクト比

端末の画面比率に合わせることで文字の歪みを防ぐ。短辺は常に 1080px 固定。

| 設定値 | 長辺 px | 備考 |
|--------|---------|------|
| `21:9` | 2520 | |
| `20:9` | 2400 | AQUOS sense4 plus 等 |
| `16:9` | 1920 | **デフォルト** |
| `4:3`  | 1440 | |

#### カラーテーマ

| 値 | 説明 |
|----|------|
| `auto` | OS の ダーク/ライト 設定に自動追従（`prefers-color-scheme` 監視） |
| `default` | ダークテーマ（寒色） |
| `light` | ライトテーマ（白背景） |
| `campfire` | 焚き火 |

> **警告表示**: 「有機EL」＋「ライトテーマ」を同時選択した場合、設定モーダル内に警告ボックスを赤字で表示。

### 項目設定

表示/非表示をユーザーが任意に設定可能。

- `秒を表示`
- `天候情報全体`（OFF にすると以下がグレーアウト・操作不能）
  - `体感温度` / `湿度` / `風速` / `UV指数` / `降水量` / `気圧`

### 天気設定

- **WeatherAPI APIキー**: 未設定時はアプリ起動から 1 秒後に自動で設定画面を開く。
- **取得場所**: 都市名（例: `Sapporo`）または緯度経度。

### クレジット・リセット

- GitHubリポジトリリンク（`nekoryu/clockck`）
- WeatherAPI.com クレジット
- **アプリの強制リセット**: `localStorage` 全消去 + ServiceWorker 登録解除 + キャッシュストレージ削除 → リロード。確認ダイアログあり。

## ハードウェア保護・省エネ制御

### 1. 有機EL 焼き付き防止（OLEDモード）

- **ピクセルシフト**: 1 分ごとに `datetime-area` と `weather-area` を ±200px のランダム量で `translate()` アニメーション（2 秒・ease-in-out）。
- **自動セーフエリア確保**: OLEDモード時、スケール計算に `±200px × 2 = 400px` の余白を加算し、シフト後も要素が画面外にはみ出さないことを保証。

### 2. 環境光センサー連動

`AmbientLightSensor` 対応環境において、照度が `10 lux` 未満になると `.dim` クラス（輝度 60% ＋ 背景黒化）を適用して眩しさを低減。非対応環境では本機能は動作しない。

### 3. Screen Wake Lock

`Wake Lock API` によりスクリーンの自動消灯を抑制。PWA スタンドアロン起動時および フルスクリーン時に有効化。フルスクリーン解除時（非PWA）に解放。

---

## PWA 仕様

- `manifest.json` によりホーム画面へのインストールに対応。
- ServiceWorker (`sw.js`) によりオフラインキャッシュを実現。
- PWA 起動時（`display-mode: standalone / fullscreen`）はフルスクリーンボタンを非表示。
