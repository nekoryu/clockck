# 仕様

## デバイス仕様
- **ターゲットデバイス**: AQUOS sense4 plus
- **画面解像度**: フルHD+（2400×1080）相当のアスペクト比
- **想定環境**: 横向き（ランドスケープ）表示

## 表示・レイアウト方針
- **解像度非依存の設計**: 端末ごとのDPI設定の差異による影響を防ぐため、ピクセル数への依存は避け **アスペクト比 2400:1080** を仮想的な基本解像度とする。
- **フルストレッチ表示**: 画面全体（2400:1080の領域）に隙間なく目一杯キャリブレーションして表示する。
- **段組レイアウト**:
  - 時刻表示（左側）と日付・年号表示（右側）のブロック高さを完全に一致させる。
  - 日付の上に、西暦および和暦（元号）を表示する。和暦は `Intl.DateTimeFormat` による自動算出。
  - 折り返しを禁止し、常に横一列の美しい整列を維持する。

## 技術スタック
- **コア**: HTML5 / CSS3 / JavaScript (ES6+)
- **依存ライブラリ**: なし（Zero Dependency）。Bootstrap, jQuery 等を排し、極限まで軽量化。
- **フォント**: Google Fonts (`M PLUS 2`) のみに最適化。

## PWA (Progressive Web App) 対応
- **インストール可能**: ホーム画面に追加してスタンドアロンアプリとして動作。
- **オフライン動作**: Service Worker により、一度読み込めばネットワーク不要で動作可能。
- **全画面表示**: アプリ起動時はブラウザのUIを排除した `fullscreen` 表示をサポート。

## 安定化・不具合対策（実機最適化済み）
- **文字巨大化防止**: Android Chrome の `text-size-adjust` を制御し、フォントの自動拡大を抑制。
- **スケーリング安定化**: 起動直後のビューポート不安定を考慮し、多段計算ロジックを搭載。
- **全画面バグ回避**: PWA起動時とブラウザ閲覧時を判定し、`requestFullscreen()` の重複リクエストによるブラックアウトを防止。

## 開発・テスト
- **一時ローカルサーバーの起動**:
  ```bash
  python -m http.server 8000
  ```
- **URL**: `http://[IP_ADDRESS]:8000`
