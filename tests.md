# clockck テスト観点一覧

> ローカルサーバー起動後（`_ローカルサーバー開始.bat`）に Chrome DevTools のデバイスエミュレーターや実機で確認する想定。

---

## 1. 表示スケーリング（基本）

| # | 観点 | 手順 | 期待結果 |
|---|------|------|----------|
| 1-1 | 横画面でのスケール | ブラウザを横長にリサイズ | `main` がアスペクト比を維持して画面内に収まり、文字が歪まない。余白は黒背景で中央揃え |
| 1-2 | 縦画面でのスケール | ブラウザを縦長にリサイズ | `main` がアスペクト比を維持して画面内に収まり、文字が歪まない。余白は黒背景で中央揃え |
| 1-3 | リサイズイベント追従 | ウィンドウをリアルタイムにドラッグリサイズ | サイズ変化に追従してスケールが更新される |
| 1-4 | orientationchange 追従 | 実機で画面回転 | 回転後にレイアウトが正しく再計算される |
| 1-5 | はみ出し防止 | ベース比率と実画面比率が異なる任意の組合せ | コンテンツが画面外にはみ出さない（短辺基準 `min(scaleX, scaleY)` を使用） |

## 2. アスペクト比手動選択

| # | 観点 | 手順 | 期待結果 |
|---|------|------|----------|
| 2-1 | デフォルト値 | 初回起動（またはキャッシュ消去後） | アスペクト比が `16:9` で設定されている |
| 2-2 | 21:9 選択 → 保存 | 設定画面で `21:9 (7:3)` を選択し保存 | 横: base=2520×1080、縦: base=1080×2520 で scale 計算。はみ出しなし |
| 2-3 | 20:9 選択 → 保存 | 設定画面で `20:9` を選択し保存 | 横: base=2400×1080、縦: base=1080×2400 で scale 計算。はみ出しなし |
| 2-4 | 16:9 選択 → 保存 | 設定画面で `16:9` を選択し保存 | 横: base=1920×1080、縦: base=1080×1920 で scale 計算。はみ出しなし |
| 2-5 | 4:3 選択 → 保存 | 設定画面で `4:3` を選択し保存 | 横: base=1440×1080、縦: base=1080×1440 で scale 計算。はみ出しなし |
| 2-6 | 設定の永続化 | アスペクト比を変更して保存 → ページリロード | リロード後も同じアスペクト比が選択・適用されている |
| 2-7 | 異比率でのはみ出しなし確認 | `4:3` を設定して 20:9 画面（2400×1080）で表示 | `min(scaleX, scaleY)` により content が縦にフィット、横に黒余白が表示されはみ出しなし |
| 2-8 | UIプルダウンの選択肢 | 設定画面を開く | `21:9 (7:3)` / `20:9` / `16:9` / `4:3` の4選択肢が存在する |

## 3. OLEDモード（焼き付き防止）

| # | 観点 | 手順 | 期待結果 |
|---|------|------|----------|
| 3-1 | OLEDモード有効化 | 設定で「有機EL」を選択し保存 | `main` がはみ出し防止用マージン（±200px = 400px）分を加味した縮小スケールになる |
| 3-2 | ピクセルシフト発動 | OLEDモード有効で1分放置 | 時刻・天気エリアがランダムな方向に2秒かけてスムーズに移動する |
| 3-3 | シフト範囲 | DevTools で `transform` の値を確認 | `translate(Xpx, Ypx)` の X・Y がそれぞれ −200〜200px の範囲内 |
| 3-4 | OLEDとアスペクト比の連動 | アスペクト比を変更した状態でOLEDモードも有効 | 新しいベースサイズ + 400px の余白でスケールが計算される |
| 3-5 | OLEDモード無効化 | 「有機EL」から「液晶」に切り替えて保存 | ピクセルシフトが停止し、位置が (0, 0) に戻る |
